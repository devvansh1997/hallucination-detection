\subsection{Hallucination Rate as a Predictor of Full-Beam Gain}

Full-beam always dilutes the hallucination rate (adds more correct than
hallucinated beams from known prompts).  Whether this helps or hurts
depends on the pre-dilution truthful beam count.

\caption{Hallucination rate dilution vs.\ AUROC gain.  Full-beam functions as
         a data-augmentation strategy: it disproportionately helps truthful-poor
         datasets (Qwen NQ-Open: only 353 truthful beams under early-stop, 1,138
         under full-beam; AUROC jumps 20.2\%).  Truthful-rich datasets
         (TyDiQA: already $>$3,700 truthful beams) gain little or regress.}
\end{table}

\noindent
\textbf{Key insight:} Full-beam acts as over-sampling for the minority class.
When the truthful beam count is very low ($<$1,000), adding 3--10$\times$ more
truthful examples significantly stabilises the HOSVD subspace.  When truthful
beams already number in the thousands, additional copies are redundant and the
diluted hallucination signal hurts the classifier.

\section{Evaluation Protocol}

We replicate HARP's exact evaluation protocol to ensure fair comparison:

\begin{enumerate}
  \item \textbf{Beam Search Generation.} For each prompt, the model generates
        10 candidate answers via beam search (\texttt{num\_beams=10},
        \texttt{num\_return\_sequences=10}).  Early stopping halts further
        beam scoring once a correct answer is found, reducing computational
        overhead on known prompts.

  \item \textbf{Contrastive Judging (BLEURT + ROUGE-L).} Each generated beam
        is scored against \emph{both} the correct reference answer(s) and
        any provided incorrect answer(s) using BLEURT-20 (semantic similarity)
        and ROUGE-L (lexical overlap).  A beam is classified as \emph{correct}
        if: $\text{BLEURT} > 0.5 \lor \text{ROUGE-L} > 0.7$ (contrastive
        advantage check always passes with threshold $-1.0$).  This rubric
        matches HARP's \texttt{DatasetJudge} implementation.

  \item \textbf{Known/Unknown Classification.} A prompt is labeled \emph{known}
        if \textbf{any} of its 10 beams is deemed correct; otherwise it is
        \emph{unknown} (all beams hallucinated).  Known prompts contribute
        both truthful and hallucinated beams; unknown prompts contribute
        only hallucinated beams.

  \item \textbf{Zero-Leakage Train/Valid Split.} Known prompts are split
        75/25 for training and validation.  All unknown prompts are added
        to the validation set \emph{only} — the training set never sees a
        single unknown-prompt beam.  This guarantees that no hallucination
        signal leaks into the HOSVD factor matrix computation.

  \item \textbf{Population HOSVD Projection.} Factor matrices $U_L \in
        \mathbb{R}^{L \times 5}$ and $U_D \in \mathbb{R}^{D \times 64}$
        are computed exclusively from the \emph{training partition} via the
        chunked Gram matrix trick.  Both training and validation beams are
        projected: $\mathcal{G} = U_L^\top \, H_{\text{pool}} \, U_D$
        yielding a $(5, 64)$ core tensor per beam, flattened to 320 features.

  \item \textbf{Random Forest Classification.} A Random Forest classifier
        (200 trees, balanced class weights, \texttt{random\_state=42}) is
        trained on the 320-dimensional projected training features and
        evaluated on the combined validation set (25\% of known-prompt
        beams + 100\% of unknown-prompt beams).  AUROC is computed via
        scikit-learn with hallucination as the positive class.

  \item \textbf{No Multiple Sampling.} Unlike HARP and HaloScope (which
        require 10+ independent forward passes), our HOSVD features are
        extracted from a single generation pass per prompt.  The 10-beam
        search serves only the evaluation rubric, not the feature extraction.
\end{enumerate}

\noindent
\textbf{Key Observations:}
\begin{enumerate}
  \item \textbf{HOSVD is the second-best method on 6 of 8 model--dataset pairs.}
        On Qwen, HOSVD ranks 2nd on TruthfulQA (78.2), TriviaQA (86.2), and
        TyDiQA (81.4).  On LLaMA, HOSVD ranks 2nd on NQ-Open (83.3),
        TruthfulQA (79.6), and TriviaQA (76.3).  HARP claims \#1 on all 8
        pairs.  All HOSVD values use the full 10-beam protocol.

  \item \textbf{On Qwen-2.5-7B, HOSVD outperforms all non-HARP baselines}
        on TruthfulQA (78.2 vs.\ 63--64), TriviaQA (86.2 vs.\ 76--85),
        and TyDiQA (81.4 vs.\ 30--75).  NQ-Open (73.5) trails EigenScore
        (78.9) by 5.4 points.

  \item \textbf{Cross-model consistency:} TruthfulQA AUROC is nearly identical
        across models (77.2 vs.\ 77.4), suggesting the HOSVD subspace captures
        a model-agnostic hallucination signature.

  \item \textbf{No multiple sampling required.} Unlike HARP and HaloScope
        (marked ``Single'' in HARP Table~1), HOSVD extracts features from
        a single forward pass; the 10-beam generation is only for the
        HARP-compatible rubric, not for feature extraction.

  \item \textbf{Compression efficiency.} HOSVD compresses raw $(L, T, D)$
        hidden states (up to $32 \times 10 \times 4096 \approx 1.3$M entries)
        into a 320-dimensional core vector ($\sim$4000$\times$ reduction)
        while retaining discriminative power.
\end{enumerate}
